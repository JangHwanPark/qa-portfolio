# 1단계 · 요구사항 분석 (모범답안)

[기능 명세](../README.md#기능-명세-v10) 를 5가지 기준으로 분해한 예시.
본인 답안을 별도 파일/노트에 먼저 작성한 뒤 본 문서와 비교 학습한다.

## 5가지 분석 기준
1. 사용자
2. 주요 기능
3. 상태 변화
4. 예외 상황
5. 외부 시스템 또는 정책 조건

## 작성 시 체크리스트
- [x] 시간 조건(30분 / 90일 / 30일) 항목화
- [x] 횟수 임계값(5회) 명시
- [x] 자동 처리(잠금 자동 해제 / 휴면 자동 전환) 분리
- [x] 사용자 분기(활성 / 잠금 / 휴면 / 미가입) 모두 포함
- [x] 메시지 정책(사용자 열거 방지) 별도 항목

---

## 1. 사용자
- 정상 회원 (활성 상태)
- 미가입자 (등록되지 않은 이메일 사용자)
- 잠금 계정 사용자 (실패 5회 누적 후 30분 미경과)
- 휴면 계정 사용자 (90일 미접속)
- 비로그인 방문자 (재설정 메일 요청 가능)

## 2. 주요 기능
- 이메일 + 비밀번호 입력 및 검증
- 로그인 시도 (세션 발급)
- 자동 로그인 옵션 (30일 세션 유지)
- 비밀번호 재설정 메일 요청
- 휴면 해제 안내 진입

## 3. 상태 변화
- 비로그인 → 로그인 (검증 성공)
- 활성 → 잠금 (실패 5회 연속)
- 잠금 → 활성 (잠금 후 30분 경과, 자동)
- 활성 → 휴면 (90일 미접속, 자동)
- 휴면 → 활성 (휴면 해제 절차 완료)
- 실패 카운터: 0 → 1 → ... → 5 (성공 시 0으로 리셋, 추측)

## 4. 예외 상황
- 등록되지 않은 이메일
- 비밀번호 불일치 (1~4회): 일반 실패 메시지
- 비밀번호 5회째 실패: 잠금 전환
- 잠금 상태에서 정답 입력: 로그인 차단 (잠금 우회 금지)
- 휴면 계정 로그인 시도: 휴면 해제 안내로 분기
- 빈 값 / 이메일 형식 오류 / 비번 길이 미달
- 비밀번호 재설정 메일 발송 실패 (외부 메일 시스템 장애, 추측)
- (추측) 동시 다중 기기 로그인 시 기존 세션 처리 정책 미정

## 5. 외부 시스템 또는 정책 조건
- 비밀번호 정책: 8~20자, 영문+숫자+특수문자 필수
- 잠금 정책: 실패 5회 / 잠금 30분
- 휴면 정책: 90일 미접속
- 세션 정책: 일반 세션 만료 vs 자동 로그인 30일 (서버 정책)
- 메일 발송 시스템 (재설정 메일)
- 메시지 정책: 이메일 존재 여부를 응답으로 노출하지 않음 (사용자 열거 공격 방지)
- (추측) 봇 차단 (reCAPTCHA 등)

---

## 학습 포인트

- **임계값은 모두 숫자로**: "30분", "5회", "90일", "30일", "8~20자" 같은 수치가 항목화되어야 누락 없이 TC 로 풀어낼 수 있다.
- **자동 처리는 별도**: "잠금 30분 후 자동 해제", "90일 미접속 자동 휴면 전환" 처럼 사람이 트리거하지 않는 흐름은 따로 잡아야 한다.
- **메시지 정책은 보안 항목**: 미가입 vs 비번 오류를 다른 메시지로 노출하면 계정 열거 공격이 가능하다. "동일 메시지" 라는 명세는 보안 정책으로 분류해야 의도가 잡힌다.
- **(추측) 표기**: 명세에 없지만 합리적으로 가정한 항목(reCAPTCHA, 다중 기기 세션)은 `(추측)` 으로 분리한다. 본인 추측을 사실처럼 섞으면 평가에서 감점.
