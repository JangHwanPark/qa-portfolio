# 1단계 · 요구사항 분석

[기능 명세](../README.md#기능-명세-v10) 를 5가지 기준으로 분해한다.

## 5가지 분석 기준
1. 사용자
2. 주요 기능
3. 상태 변화
4. 예외 상황
5. 외부 시스템 또는 정책 조건

## 체크리스트
- [ ] 입력 필드별 검증 규칙(길이 / 형식 / 문자 조합) 수치까지 항목화
- [ ] 검증 트리거 시점(실시간 vs 블러 vs 제출) 구분
- [ ] 약관 필수 / 선택 분리, 연령 확인 별도
- [ ] 이메일 상태 분기(미가입 / 가입 완료 / 인증 대기) 명시
- [ ] 시간 조건(24시간 인증 유효, 자동 폐기) 항목화
- [ ] 보안 / 법적 정책(평문 금지, 사용자 열거 방지, 연령 제한) 별도 항목

## 1. 사용자
- 정식회원
  - 이미 등록된 사용자(등록된 이메일 사용자)
- 정식회원 아님
  - 미가입자 (등록되지 않은 이메일 사용자)
  - 인증 대기상태 사용자

## 2. 주요 기능 = 시스템이 제공하는 능력/행위 단위
- 이메일 + 비밀번호 + 닉네임 + 휴대폰 번호 입력 검증
- 이메일 / 닉네임 중복검사
- 약관동의 입력 검증
- 회원가입 제출 / 이메일 인증 요청
- + 24시간 경과시 신청 자동 폐기(시스템 자동)

## 3. 상태 변화
### 상태정의
- 비회원: 가입 전
- 인증 대기: 가입 요청 통과, 인증 메일 발송, 토큰 발급(TTL 24h)
- 회원: 이메일 인증 완료, 로그인 가능

### 상태 전이
- 비회원 -> [가입 제출 / 검증 통과] -> 인증 대기(메일 발송, 토큰 발급)
- 인증 대기 -> 회원(인증 완료)
- 인증 대기 -> 미가입(토큰 만료)

## 4. 예외 상황
- input 공통
  - null, 빈 문자열 입력
  - sql 쿼리 입력
  - 입력 길이제한 미달/초과
- 이메일
  - 이메일 형식 미일치 (name@domain)
  - 이미 가입한 이메일로 재가입
  - 인증 이메일을 전달받지 못한 경우
  - 만료 이메일로 가입 가능 여부
  - + 인증 대기 이메일로 재가입
- 닉네임
  - 중복 닉네임 제출
- 비밀번호
  - 비밀번호 확인이 불일치
  - 특수문자 미포함
- 휴대폰 번호
  - 숫자만 입력한 경우
- 필수 체크박스 미입력
- 입력 비동기 처리
  - 유효성 검사 진행중 submit 버튼 클릭
