# 3단계 · 테스트 케이스 도출

[02_scenarios.md](02_scenarios.md) 의 시나리오를 동등 분할 / 경계값 기법으로 TC 표로 풀어낸다.

## 컬럼 정의
- **유형**: Positive / Negative / Boundary / Negative-Boundary
- **우선순위**: P0(필수) / P1(중요) / P2(보통) / P3(낮음)
- **비고**: 연결되는 SC ID 명시

## 작성 조건
- **최소 18개 이상** TC
- 동등 분할 클래스(이메일 상태 4종, 닉네임 사용 가능 / 중복, 약관 동의 조합) 커버
- 경계값 명시적 표기
  - 비밀번호 길이: 7 / 8 / 20 / 21
  - 닉네임 길이: 1 / 2 / 10 / 11
  - 비밀번호 문자 조합: 영문만 / 영문+숫자 / 영문+숫자+특수문자
- 부정 입력 케이스(빈 값 / 형식 오류 / 공백만 입력)
- 비동기 중복 확인 race 케이스(확인 중 제출)
- 한 TC = 한 검증 포인트

## 작성 후 셀프 점검

- 한 TC 에 단정이 여러 개 섞이지 않았는가
- 사전 조건이 명확해서 자동화 시 사전 상태 세팅이 결정되는가
- 기대 결과가 단정 가능한 표현인가 ("잘 동작한다" 같은 표현 금지)
- 경계값 표기에 "양쪽 경계 + 1" 이 모두 들어갔는가 (예: 7 / 8 / 20 / 21)
