# 2단계 · 시나리오 설계 / 우선순위

[01_requirements.md](01_requirements.md) 분석 결과를 출시 전 검증 시나리오로 풀어낸다.

## 컬럼 정의
- **우선순위**: P0(머스트, 출시 차단) / P1(중요) / P2(보통) / P3(여유 시)
- **분류**: Smoke / Sanity / Regression / Exploratory / Performance / Security

## 작성 조건
- **최소 12개 이상** 시나리오
- 입력 검증 시나리오(이메일 / 비번 정책 / 닉네임 / 휴대폰) 모두 포함
- 약관 필수 미동의 시나리오 포함
- 이메일 상태별 분기(미가입 / 가입 완료 / 인증 대기) 시나리오 포함
- 비기능 시나리오(접근성 / 키보드 Tab 순서) **최소 1개 이상**
- 보안 시나리오(사용자 열거 방지 / 평문 노출 금지) **최소 1개**
- 매뉴얼 / 자동화 어느 쪽이 적합한지 비고에 표기
- 우선순위 분포 가이드: P0 20% 내외, P1 30~40%, P2 30%, P3 10~20%

## 작성 후 셀프 점검

- P0 이 "출시 차단" 수준만 모여 있는가 (편하게 P0 부여 X)
- 회귀 영역(예: 가입 해피패스 / 약관 필수 강제)을 빠뜨리지 않았는가
- 매뉴얼이 적합한 시나리오(트리거 시점 / 로딩 표시)와 자동화가 적합한 시나리오(검증 메시지 / 경계값)가 명확히 갈리는가
