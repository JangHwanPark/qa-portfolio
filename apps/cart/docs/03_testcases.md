# 3단계 · 테스트 케이스 도출

[02_scenarios.md](02_scenarios.md) 의 시나리오를 동등 분할 / 경계값 기법으로 TC 표로 풀어낸다.

## 컬럼 정의
- **유형**: Positive / Negative / Boundary / Negative-Boundary
- **우선순위**: P0(필수) / P1(중요) / P2(보통) / P3(낮음)
- **비고**: 연결되는 SC ID 명시

## 작성 조건
- **최소 15개 이상** TC
- 동등 분할 클래스(일반 / 재고 한정 / 1인 최대 / 품절) 모두 커버
- 경계값 명시적 표기
  - 수량 0 / 1 / 최대 / 최대 + 1
  - 재고 한정 상품(B): 재고 직전 / 직전+1 / 직전+2 / 초과
  - 1인 최대 상품(C): 4 / 5 / 6
- 직접 입력 부정 케이스(음수 / 0 / 문자 / 소수 / 빈 값)
- 한 TC = 한 검증 포인트

## 작성 후 셀프 점검

- 한 TC 에 단정이 여러 개 섞이지 않았는가
- 사전 조건이 명확해서 자동화 시 사전 상태 세팅이 결정되는가
- 기대 결과가 단정 가능한 표현인가 ("잘 동작한다" 같은 표현 금지)
