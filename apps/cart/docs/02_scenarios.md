# 2단계 · 시나리오 설계 / 우선순위

[01_requirements.md](01_requirements.md) 분석 결과를 출시 전 검증 시나리오로 풀어낸다.

## 컬럼 정의
- **우선순위**: P0(머스트, 출시 차단) / P1(중요) / P2(보통) / P3(여유 시)
- **분류**: Smoke / Sanity / Regression / Exploratory / Performance / Security

## 작성 조건
- **최소 10개 이상** 시나리오
- 수량 정책(재고 한도 / 1인 최대) 검증 시나리오 포함
- 비기능 시나리오(성능 / 접근성) **최소 1개 이상**
- 매뉴얼 / 자동화 어느 쪽이 적합한지 비고에 표기
- 우선순위 분포 가이드: P0 20% 내외, P1 30~40%, P2 30%, P3 10~20%

## 작성 후 셀프 점검

- P0 이 "출시 차단" 수준만 모여 있는가 (편하게 P0 부여 X)
- 회귀 영역(예: 단순 수량 증감 시 금액 정확성)을 빠뜨리지 않았는가
- 매뉴얼이 적합한 시나리오와 자동화가 적합한 시나리오가 명확히 갈리는가
